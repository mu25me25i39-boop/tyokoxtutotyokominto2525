GrapheneOSの導入方法

1. セキュリティ特化のAndroid

GrapheneOSとは？

GrapheneOS は Google Pixel 向けの セキュリティとプライバシー最優先のAndroid系OS。無料で導入でき、対応Pixelを用意すれば誰でも利用できる。

普段のPixelをセキュリティ特化型として使えるようになり、攻撃への耐性やアプリ権限の細かい制御が強化されるため、情報漏洩リスクを大きく減らせる。

使い方の流れは、対応Pixelを用意して 公式Web Installer で導入するだけ。PCとケーブルを繋いでブラウザの指示に従うだけで、専門知識なしに安全に導入できます。これでセキュリティ強化済みのPixelとして使い始められる。

※導入時は初期化やブートローダー操作を伴い、データは消去されます。バックアップ必須であり、手順ミスによるリスクも理解して進めること。

GrapheneOSのメリットとデメリット

メリット

  • ゼロデイ攻撃に強い防御力
    メモリ安全性の技術やOSの核となる部分（カーネル）の保護が強化され、悪意のあるアプリがシステムの根幹に侵入するのを防ぎ、未知の攻撃にも比較的耐性がある。
  • アプリの挙動を細かく制御
    「ネット通信を禁止」「センサー利用を無効化」といった設定が可能で、情報漏洩を防げる。
  • コピー情報は自動で消去
    クリップボードの内容は一定時間で自動消去され、他アプリからの不正取得を防ぐ。
  • Googleサービスを選んで利用可能
    Google Playをサンドボックス化して使えるため、必要な場合だけ導入できる。不要なら完全に外すことも可能。
  • オープンソースによる透明性
    ソースコードが公開され、研究者や専門家による監査が可能。安全性の検証が開かれており、裏で情報を送っているようなバックドアが混入するリスクが極めて低い。
  • Pixelハードを最大限活用
    Titan Mチップやセキュアブートなど、Pixel独自のセキュリティ機能をそのまま利用できる。
  • プロファイル分離で安全に使い分け
    「仕事用」と「プライベート用」を完全に分離でき、環境を切り替えて利用可能。

デメリット

  • 利便性：Googleアシスタントが使えない（プライバシー保護のため非対応）。
  • 利便性：Pixel独自の一部機能が制限（AIカメラ処理など、Googleサーバー依存部分）。
  • 互換性：Google Playストアはそのまま使えず、「Sandboxed Google Play」や Aurora Store を利用する必要がある。
  • 互換性：一部アプリが動作しない（例：銀行アプリ、キャリア決済アプリ）。
  • 利便性：利便性よりセキュリティ重視の設計で、設定や管理に手間がかかる。
  • サポート：Pixel本体の保証は残るが、Google公式サポート対象外となる。
GrapheneOSの導入方法（概要）

必要なもの：

  • PC（Google Chrome/Chromium系推奨）
  • Pixel端末
  • 接続用ケーブル（データ通信対応）

Pixel側の準備

  • 開発者モードON → ビルド番号7回タップ
  • OEMロック解除ON
  • USBデバッグON

インストール手順

  1. PixelをFastbootモードにする
  2. PCで 公式Web Installer を開く
  3. 「Unlock bootloader → Download release → Flash release → Lock bootloader」を順に実行
  4. Fastbootで「Start」を選び起動 → 完了

⚠ 注意点

  • 作業中はデータ全消去
  • ケーブルを抜かない／電源切らない
  • 毎回の起動時に黄色警告は正常
  • 必ず「Lock bootloader」を最後に実行
2. GrapheneOS 導入後

プロファイル分け（メインとサブ）

GrapheneOSの強みは ユーザープロファイル分離にあります。Windowsの「ユーザーアカウント」に近いイメージです。※なお、この挙動はAndroid標準の複数ユーザー機能による仕様で、GrapheneOS独自の制限ではありません。GrapheneOSではこの標準機能を 強化されたセキュリティ基盤上で活用できる点が特徴です。

メインプロファイル（オーナー）

  • 電話番号・SMSは必ずここに紐づきます。
  • 通知は常に届き、他のプロファイルとは完全に分離されます。

サブプロファイル

  • 追加ユーザーとして新規セットアップ。アプリやデータは完全独立。
  • 通知はメインと干渉せず、開いたときのみ反応します。

メリット

  • ストレージ暗号化はプロファイル単位で強固。各ユーザーごとに暗号鍵が分かれます。
  • メインを開いた状態ではサブのデータにアクセスできず、逆も同じです。
  • サブでアプリが汚染されてもメインには一切影響がありません。

デメリット

  • 写真や動画の保存場所が異なり、共有にはクラウドやUSB転送が必要。
  • 通知は切り替え中のみ届きません。サブ利用時はメインの通知が止まります。
  • サブを閉じると、そのプロファイルのアプリは 完全に停止します（音楽再生やダウンロードも継続しません）。理由：安全のため完全分離する仕組みだからです。
  • メインとサブを同時に動かすことはできません。

【補足】通知の仕様について

GrapheneOSには、サブプロファイルの通知をメインプロファイルに転送する機能はありません。これはプロファイルの完全な分離を維持するための設計思想です。

作成方法

設定 → システム → 複数のユーザー → ユーザーを追加。

実用例

  • メイン：仕事・連絡系アプリのみ
  • サブ：ゲームや動画アプリを集中して入れる

切り替え方法

通知バー右上のユーザーアイコン → プロファイル選択 → パスコード入力。

※補足：独立した環境でも、物理的なストレージは共有。

Google Playストアをどう使うか？

GrapheneOSには標準でGoogle Playストアが含まれていません。代わりに Sandboxed Google Play を導入することで、ほぼ通常のAndroidと同じようにPlayストアを利用できます。

Sandboxed Google Play

  • メリット：普通のAndroidと同じようにアプリを入れられる。
  • デメリット：Google関連のプロセスが動作するため、わずかにプライバシー妥協がある。

Aurora Store

Googleアカウント不要でPlayストアのアプリをダウンロードできる代替ストア。

実用的な使い分け

  • メイン：Googleなしで軽量運用（Aurora）
  • サブ：Google前提アプリ（Sandboxed）

通知の運用ポイント

  • メインに置くべき：電話・SMS・LINE
  • サブに隔離すべき：SNS・ゲーム・銀行アプリ

導入方法

Sandboxed Google Play：

  • App Storeから「Google Play services」「Google Play Store」をインストール。

Aurora Store：

  • Vanadiumで 公式ページ を開く。
日本語入力を有効にする（Gboard導入）

GrapheneOSには標準で日本語入力がありません。

導入方法

  • Sandboxed Google Play：Gboardをインストール。
  • Aurora Store：Googleアカウント不要。

有効化手順

  1. 設定 → システム → 言語と入力
  2. オンスクリーンキーボード → 管理
  3. Gboard をオン
  4. 地球マーク長押し → 日本語入力
セキュリティソフトとVPN

セキュリティソフト

  • 原則不要：標準セキュリティが強固。
  • デメリット：電池消費・動作の重さ。
  • 例外：サブプロファイルで補助的に使う場合のみ。

VPN

公共Wi-Fi利用が多い人は必須級。

まとめ

  • VPNは全ユーザーに推奨。
  • セキュリティソフトはサブのみ。
Pixelに戻す方法

注意

この手順では端末内のすべてのデータが消去されます。

復旧方法

Fastbootモードで再度 flash-all を実行すれば復旧できる場合が多い。

戻す方法（簡略版）

  1. Google公式ファクトリーイメージをダウンロード。
  2. PixelをFastbootモードにする。
  3. flash-all を実行。
  4. 起動後Googleログイン。

必要なもの

  • USBケーブル
  • PC
  • Pixel端末
  • Platform Tools
  • Factory Image

Pixel端末側の準備

  1. ビルド番号7回タップ → 開発者向けオプションON
  2. OEMロック解除ON
  3. Fastbootモードで起動
パソコン側の準備（ダウンロード＆展開）

Platform-Tools と Factory Image を同じフォルダにまとめる。

アンロック & フラッシュ手順
  1. PixelをPCに接続。
  2. Shift+右クリック → ターミナルで開く。
  3. fastboot devices
  4. fastboot flashing unlock
  5. flash-all を実行。
  6. 再起動後、必要なら fastboot flashing lock。
アンロック & フラッシュ手順（詳細）
  1. USBケーブルでPixelをPCに接続。
  2. フォルダの空き領域で Shift + 右クリック → 「ターミナルで開く」 を選択。
  3. 認識確認： .\fastboot devices
  4. ブートローダーのアンロック： .\fastboot flashing unlock
    → 端末に「Unlock bootloader?」が出るので Yes を選び電源ボタンで決定。
  5. 同フォルダ内の flash-all.bat（macOS/Linuxは flash-all.sh）を実行。 → ROMの書き込みがスタート。
  6. Pixelが自動で再起動し Fastboot画面に戻る。 Fastboot画面で「Start」に合わせて電源ボタンで確定 → 起動したら初期セットアップ（Googleアカウントでログイン）。
  7. セキュリティを元に戻すため、再度 Fastboot で起動し fastboot flashing lock を実行して ブートローダーを再ロック。

    注意：後から再ロックすると再度初期化（データ全消去）が必要。 Pixelに戻した直後に実行するのがベスト。

  8. ケーブルは flash-all 完了後 なら抜いてOK。PC側のウィンドウも閉じて問題ありません。
